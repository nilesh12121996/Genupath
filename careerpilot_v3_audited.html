Faults & improvements — click any row to expand

1

Phase 1 & 2 shown as parallel (fault)

The original routes Agent 0 to both phases simultaneously, implying they run in parallel. This is wrong — Phase 2 needs the structured JD JSON from Phase 1 and can only start after the H1B gate passes. Fixed with a Phase 1 → Phase 2 sequential arrow.

2

Human behavior engine as a sequential step (fault)

Placed linearly between Phase 2 and Phase 3, the original implies it only activates at submission. It must be a cross-cutting service injected into Agent 1 (scraping), Agent 4 (submission), and Agent 5 (LinkedIn). Fixed with explicit "cross-cutting" labeling.

3

Phase 3 & 4 shown as parallel (fault)

Phase 4 can only begin after Phase 3 emits application.submitted. They are event-driven sequential, not parallel siblings. The original shows them as independent branches starting from the behavior engine. Fixed with a Phase 3 → Phase 4 dependency arrow.

4

LinkedIn People Search API not publicly available (caution)

This API requires LinkedIn Partner Program approval — unavailable to individual developers. Agent 5 must use a dedicated throwaway account at <8 profiles/hour as the primary path. Official API is aspirational. Never run this on your real personal account.

5

Secrets vault scoped only to Agent 0 (caution)

HashiCorp Vault shown inside Agent 0 setup implies it only exists at init time. It must be a persistent always-on infrastructure service that every agent calls at runtime for short-lived tokens. Moved to the Data Layer in this redesign.

6

Add Agent 7: Interview prep (improvement)

After an interview is booked, Agent 7 uses Pinecone memory plus the original JD to generate role-specific practice questions and model answers. Closes the loop from application to preparation and fully leverages the Pinecone investment already in the stack.

7

Add rate limiter + interview notifier (improvement)

Explicit per-platform daily caps (15 on LinkedIn, 20 on Indeed) prevent velocity-based account flags. A notification layer that detects incoming interview invite emails and fires an SMS/push alert turns the system into a real-time response tool, not just an auto-applier.

8

Specify LangGraph + Redis Streams (improvement)

The original says "State Machine" generically. LangGraph as the orchestrator and Redis Streams as the message queue makes implementation choices concrete. Also added: dead-letter queue and event replay to the message queue for fault-tolerant async agent communication.
