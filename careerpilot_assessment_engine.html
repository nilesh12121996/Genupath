End-to-end flow
COMPANY SETUP
Company uploads 5–10 seed questions per topic
Each seed question defines: the concept being tested, the difficulty level, the correct answer, and 3 plausible distractors. Think of them as templates — not the final questions.
AI ENGINE
CareerPilot generates a unique variant per candidate
Claude generates a structurally equivalent question with different numbers, scenarios, companies, variable names, or narrative context. The tested concept and difficulty are preserved. The surface question is unique — untestable on Google, unshareable on Reddit.

Core Technology

The question rotation engine

Companies provide seed questions. The AI generates structurally equivalent variants. The concept, difficulty, and correct answer type are preserved. The surface presentation is unique per candidate. No two candidates ever see the same question.

How AI generates variants — with examples
Data Structures Medium difficulty
Concept: Time complexity of nested loops
Seed question (company-provided — never shown to candidates)
What is the time complexity of the following code?
for i in range(n):
  for j in range(n):
    print(i + j)
Candidate A's variant (auto-generated)
A warehouse management system processes inventory. For each of the n warehouse sections, it checks each of the n items within it and logs the combined section-item index. What is the Big-O time complexity of this operation?
Same concept: O(n²) nested iteration — different narrative, same correct answer
Candidate B's variant (auto-generated)
Consider a matrix of size n×n. An algorithm visits every cell exactly once, performing a constant-time calculation at each cell. What is the time complexity of this algorithm?
Same concept: O(n²) — framed as a matrix problem, identical tested knowledge
Candidate C's variant (auto-generated)
A music streaming service compares every song in a playlist of length n against every other song to detect duplicates. How does the runtime scale with the size of the playlist?
Same concept: O(n²) — real-world scenario, no code shown, tests conceptual understanding
Product Sense Medium difficulty
Concept: Defining success metrics for a feature
Seed question (company-provided)
You launch a new notification feature on your mobile app. How would you measure whether it was successful?
Candidate A's variant
Your team just shipped a "Recently Viewed" shelf on a video streaming homepage. The PM asks you to define success for this feature at the end of the first week. Walk through the metrics you'd prioritise and explain what a successful result would look like.
Same concept: north star metric selection, leading vs lagging indicators — different product context
Candidate B's variant
A fintech app introduces an in-app budget tracking tool that users must opt into. Three months post-launch, the product team needs to present success metrics to the board. What would you measure and why?
Same concept: success metrics — adds complexity of opt-in vs default and long time horizon
What the engine changes vs preserves
DimensionRuleExample
Concept testedAlways preservedO(n²) complexity stays O(n²)
Correct answerAlways preservedRight answer is always the same type
Difficulty levelAlways preservedMedium stays medium ±5%
Question typeAlways preservedMCQ stays MCQ, code stays code
Surface narrativeAlways changedWarehouse, matrix, music — rotated
Variable namesAlways changedn, m, k, count — rotated
Numbers / valuesAlways changedDifferent numerical examples
Distractor optionsAlways changedPlausible wrong answers re-generated
Option orderAlways shuffledA,B,C,D randomly reassigned
Validation step: Every AI-generated variant is validated by a secondary Claude call that checks: (1) the concept being tested is identical, (2) the correct answer is preserved, (3) the difficulty delta is < 0.1 on a 0–1 scale. Variants that fail are regenerated. Companies can also flag bad variants from their dashboard.
Edge case — coding questions: For code-based questions, the engine changes the domain (e.g. sort transactions → sort sensor readings) but keeps the algorithmic requirement identical. The function signature changes, the expected output type changes, but the algorithm the candidate must implement does not.
Question bank growth: The engine generates and caches 50 variants per seed question at setup time. These are pre-validated and drawn from sequentially. When a company's variant pool is 80% used, the engine regenerates the next batch. The company never runs out of unique questions.

Feature Design · Internal Transfer

Internal employees skip the test.
Here's exactly how.

An employee of 4 years applying for a promotion should not sit next to an outside candidate in a screening test designed to filter unknowns. But the bypass must be verified — it can't be self-declared, or anyone could claim it.

The bypass flow — step by step
Employee finds role on CareerPilot
Sees the role posted publicly. Application page shows "Are you a current employee?"
→
Enters corporate email
Types their @company.com email. System checks domain against the company's verified domain list.
→
OTP sent to corporate inbox
6-digit code sent to their company email. Verifies they actually have access to that inbox today.
→
Assessment skipped
Application proceeds directly to CV review. Internal badge visible to recruiter. HR can rescind if needed.
What internal candidates see instead of the test
Internal transfer application view

The internal candidate's application shows different fields and signals:

✓Assessment skipped — verified current employee
✓Internal profile linked — your performance review data (if company consents to share with recruiter) is available
✓Manager endorsement — optional field to add a note from your current manager
iInternal probability shown — you still see the "% likely to be filled internally" signal (which now includes you)
What recruiters see for internal candidates

Internal candidates appear in a separate section of the recruiter dashboard with distinct indicators:

INTERNALClearly badged — never mixed invisibly into external pool
↑Tenure signal — years at company, team, current role visible
≡No assessment score — ranked separately, not directly compared with external scores
!HR rescind button — if the employee has an active PIP or the bypass is contested, HR can flag it
Edge cases handled
Contractor / agency staff

Contractors with @company.com emails: HR sets a flag in the company admin saying whether contractors are eligible for the bypass. If no flag set, contractors take the assessment. Company decides — CareerPilot enforces.

Employee leaves before application reviewed

The bypass is validated at time of application, not at time of review. If the employee's email access is revoked before the recruiter sees their application, the internal badge is updated to "Former employee — verify" and recruiter is notified.

Same company, different country/entity

An employee of Stripe UK applying for a Stripe US role. Company admin can configure whether cross-entity transfers qualify for the bypass. Default is yes if same email domain. Cross-entity bypass can require additional manager approval step.

Feature Design · Honesty Signal

Tell candidates the truth
before they invest time.

Some roles are posted publicly as a formality — the company already has a strong internal candidate. Candidates spend hours tailoring applications for roles that were effectively filled before the job went live. This feature surfaces that signal honestly.

How the internal hire probability signal is computed
Signal sources (in priority order)
1
Internal applications received (strongest signal)
If 3+ internal candidates have already applied and the role opened <48 hours ago, internal probability is very high.
2
Company's historical internal hire rate
Computed from past roles on CareerPilot: what % of their roles were filled by internal candidates? Varies wildly by company culture.
3
Role type & seniority
Director+ roles are filled internally 72% of the time across industries. Individual contributor roles: 28%. Highly predictive.
4
Outcome data from similar past roles (platform)
When a role closes, the hired candidate's source is logged. Over time we know: "Software roles at this company in NYC are filled internally 61% of the time."
5
Job description language patterns
Phrases like "familiarity with our internal systems" or "experience with our existing codebase" are statistically associated with roles that were pre-filled. NLP flag on JD text.

Legal design note: We never state a role IS filled internally — only that there IS a probability. All language is probabilistic ("estimated likelihood"). We do not expose individual internal candidate names or data. The signal is based on aggregate statistics and historical patterns — the same legal ground as Glassdoor salary estimates or Zillow Zestimates. Companies can opt out of sharing internal application volume, in which case the signal falls back to historical data only.
Company controls over this feature
Full transparency mode (default)

Internal application count + historical data + JD signals all feed the estimate. Candidates get the most accurate picture. Recommended — companies that use this see higher quality external applications because only motivated candidates apply.

Historical-only mode

Real-time internal application count is not shared. Signal uses only historical hire patterns and JD language. Less accurate but acceptable for companies with sensitive hiring pipelines.

Signal hidden

Company opts out entirely. No internal hire signal shown on their roles. Available but discouraged — we show companies the data that their conversion rates are better with transparency enabled.

Integrity Architecture

Seven layers of assessment integrity

The question rotation engine eliminates sharing. These layers handle everything else: AI tools, proxy test-takers, impersonation, and timing manipulation.

Anti-cheat layers
Question rotation (primary)
Every candidate gets a unique question. There is nothing to share on Glassdoor, Reddit, or Discord because no one else will ever see the exact same question. This is the single most powerful integrity mechanism.
Tab-switch & focus detection
If the browser tab loses focus for more than 8 seconds, a warning is logged. On the 3rd warning, the assessment is paused and the company is notified. The candidate can re-enter but it's flagged for recruiter review.
Adaptive timing analysis
We know the expected time distribution for each question (from calibration runs). Answers completed in under 20% of expected time are flagged as "suspiciously fast" — possible copy-paste or AI-generated response.
No copy-paste on text fields
Paste events on short-answer and code fields are blocked. Copy events are logged. The candidate can still type freely — but they can't drop in a pre-written ChatGPT answer to a short-answer question.
Keystroke entropy analysis
For code and long-form text responses: genuine typing has natural variation in inter-key timing. Pasted text shows a single large insertion event. We measure keystroke entropy and flag robotic patterns consistent with AI generation.
Identity consistency (optional)
For high-stakes roles, companies can enable webcam snapshots at random intervals (candidate-consented, never recorded, just compared to verified photo at login). Detects substituted test-takers.
Design philosophy on AI tools: We do not try to fully block AI assistants — that battle is unwinnable. Instead, question rotation means the AI has no prior context to draw from (the question is unique). Keystroke and timing analysis flags suspicious patterns. The company ultimately decides whether a flagged assessment disqualifies the candidate or just adds a note for the recruiter. We surface signals, not verdicts.

Score portability (deferred): A future feature — if a candidate passes an assessment at Company A, and Company B uses the same question bank topic, the score can be optionally shared (with candidate consent). This is a major network effect driver: the more companies use the same topic framework, the more reusable each candidate's score becomes.
Don't build assessment before these prerequisites: Company job posting portal, candidate profile with verified identity, company admin portal with billing. Without these, there's no company to upload seeds and no authenticated candidate to take the test. Assessment is Week 9–17 of the overall platform build, not Week 1.
