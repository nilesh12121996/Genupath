Layer 1 — Event Triggers

Zero polling. Push-only. Fires in seconds.

The old design checked job boards every 15 minutes. This runs the moment a job is posted.

Gmail Pub/Sub webhook
LinkedIn + Indeed email alerts → dedicated inbox → Google Pub/Sub push → triggers pipeline in <10s
WebSub / RSS feeds
Superfeedr aggregator monitors Greenhouse, Lever, RemoteOK — pushes notification on new item
Career page watcher
Playwright stores DOM hash of /jobs pages — fires only on change, not on schedule
Vector dedup gate
Embeds incoming JD → cosine similarity check → drops reposts (similarity > 0.92) before processing
Old: poll every 15 mins = max 15-min delay
New: push-triggered = <10 second delay
Instead of polling job boards on a timer, we hook into the notification systems they already provide. LinkedIn and Indeed send job alert emails — Gmail Pub/Sub fires a webhook the instant that email arrives. You get notified of new jobs before most people even refresh the page.
Layer 2 — Security Baseline + Agent 0

Always-on security. One-time setup.

HashiCorp Vault
Dynamic short-lived tokens per agent. API keys, cookies, EEO answers. Never in env vars or DB plaintext.
GDPR + audit log
Immutable append-only audit trail. 12-month TTL on job records. EEO data isolated, never in LLM context.
Resume ingestion
Claude Sonnet parses your PDF → PostgreSQL roles/bullets/skills tables. Versioned. Re-run on any update.
Truthfulness gate
truth_constraints table stores your real companies/dates. Agent 3 cannot invent experience that isn't here.
Browser profile factory
Creates aged profiles with 6 months of history. 2-week warm-up before first real application. Defeats fingerprinting.
Market intelligence
Levels.fyi + Glassdoor salary data loaded. Auto-populates salary fields at P60. Never leaves required fields blank.
The truthfulness gate is critical — without it, Claude could hallucinate experience you don't have, which is resume fraud. The truth_constraints table acts as a hard boundary: every generated bullet point is cross-checked against your real work history before it goes anywhere.

Pixel-perfect PDF via python-texlive. ATS-safe formatting. Version ID stored for outcome tracking.

The ATS loop is where most systems fail. Instead of a home-built parser, we use the Jobscan API — the same tool real recruiters use to check resumes. The 90% target from the original design was unrealistic; 80% on Jobscan is well-calibrated and achievable in 1–2 iterations for most roles.
Cross-Cutting Service

The human behavior engine — what actually defeats detection

Anti-bot systems fingerprint 6+ signals simultaneously. You need to defeat all of them.

playwright-stealth + TLS
Masks webdriver flag, JS APIs, canvas fingerprint, WebGL renderer, AudioContext. TLS/JA3 randomization defeats Cloudflare at network layer.
Residential proxy (Webshare)
Sticky session per domain — same residential IP for entire application. Data-center IPs get flagged immediately.
Gaussian timing engine
All delays drawn from Gaussian (μ=1.4s, σ=0.5s). Business hours gate: 8am–7pm only. No weekends.
Bezier mouse + typos
Mouse curves to target with slight overshoot. Typing at 45–65 WPM. 1-in-40 chance of typo + backspace.
Aged browser profiles
6 months of warm-up history per domain. Fresh profiles are the #1 bot signal — never create new ones per run.
CAPTCHA handler
2captcha primary (~15s solve). On second failure: pause agent + Slack alert + await manual clearance.
playwright-stealth alone does not defeat Cloudflare. It patches browser JS APIs but doesn't touch the TLS handshake. The JA3 fingerprint, HTTP/2 frame ordering, and behavioral mouse entropy are what modern bot management systems actually check. You need all 6 layers working together.

Agent 5 · Networker
Proxycurl API (legal) · hiring manager classifier · Claude Sonnet DM · 3–6 min reading delay
↓
Workday adapter
Custom field mapping for Workday's SPA framework. Saga checkpoint per page — resumes if crashed.
Greenhouse / Lever / Ashby
Platform-specific DOM field mappings. ~3× more reliable than generic Computer Use alone.
Proxycurl replaces "LinkedIn People Search API" — which requires enterprise partnership and is unavailable to individuals. Proxycurl is a legitimate paid API at ~$0.01/profile with no ToS risk.
The saga pattern for Agent 4 is non-negotiable. Workday and Greenhouse forms have 4–8 pages. Without checkpointing, a crash on page 6 either double-submits or wastes the entire application. Each page state is saved — restarts pick up exactly where they left off.
Phase 4 — Follow-up & Learning

The system gets smarter with every application

Agent 6 · Follow-up
5-day DM follow-up using Pinecone context recall. Checks Gmail for interview invite first — cancels follow-up if found. Max 1 per application.
Agent 7 · Interview prep
Triggered on interview invite detection. Generates 10 STAR behavioral Qs, 5 technical Qs, company brief. Delivered as PDF + Slack.
Outcome A/B framework
PostgreSQL tracks resume_version → interview rate. Chi-square test at N=30 per variant — strategy only changes at p < 0.05.
Interview invite notifier
Gmail monitored for invite subject-line patterns. Fires immediate SMS + Slack push so you never miss a response window.
Statistical rigor matters. Switching resume strategies based on 3–4 data points is noise-driven, not signal-driven. The Chi-square gate at N=30 with p < 0.05 ensures changes are based on real signal — not coincidence.
Agent 7 closes the loop that most automation systems miss entirely. When you get an interview, the system auto-generates a personalised prep pack using the exact job description and your own past interview answers stored in Pinecone — delivered before you've even replied to schedule the call.
Layers 8 & 9 — Orchestration + Data

The nervous system & memory

LangGraph + LangSmith
State machine, conditional edges, exponential backoff + jitter. Full trace per run. Circuit breaker per external API.
Redis Streams event bus
CloudEvents schema. Consumer groups. Dead-letter queue. Event replay for debugging. Async agent communication.
Supabase PostgreSQL
Jobs, experiences, outcomes, audit log, salary benchmarks, truth constraints. Row-level security. 12-month TTL.
Pinecone vector DB
Namespaces: interview-memory, jd-archive, cover-letter-bank. Semantic retrieval for resume + dedup.
Upstash Redis
Session state cache, rate limiter token buckets, dedup bloom filter. All free-tier at this volume.
Cloudflare R2 + Vault
Versioned resume PDFs and browser profiles in R2. Vault self-hosted on VPS for secrets management.
Everything in the data layer runs on free tiers at a volume of 5–20 applications/day. Supabase free tier handles the PostgreSQL load. Pinecone free tier covers the vector storage. Upstash Redis free tier covers the cache. The only unavoidable paid cost is Claude API tokens and residential proxy bandwidth.
