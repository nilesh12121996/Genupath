🚀 CareerPilot 2026 | Final Architecture

Strategic Multi-Agent Automation Pipeline (H1B Transfer Focused)

Tier 1: The Control Plane (Brains & Memory)

🧠 LangGraph Orchestrator Core

Central state machine managing the application lifecycle. Handles task routing, error retries, and state persistence.

Responsibilities:
  • Maintains "Ground Truth" for every Job ID (Status: Found, Verified, Tailored, etc.).
  • Cycles through the ATS Simulation Loop until thresholds are met.
  • Manages high-level "Phase" handoffs between agent clusters.

🔐 Vault & Setup (Agent 0)

Secure storage for Master Resume (NJIT/Antra), EEO answers, and rotated session tokens.

Data Points:
  • Experience DB: Structured JSON of all professional history.
  • Secrets: LinkedIn cookies, Portal credentials, and USCIS API keys.
  • Identity Vault: Pre-defined diversity and disability disclosure answers.

📊 Observability Node

Monitors token budget, performance metrics, and application success rates.

Metrics Tracked:
  • Daily Token Spend (Guardrail alerts on budget breach).
  • Agent response times and LangSmith traces.
  • Geographic success rate (Concentration on Herndon, VA and surrounding areas).
Tier 2: The Agent Pipeline (Specialized Workers)

🔍 Phase 1: Sourcing Scout

Event-driven discovery via RSS/Webhooks. Extracts Job Descriptions into structured JSON via GPT-4o-mini.

  • Triggers on change, not on cron (Zero-polling architecture).
  • Extracts Technical Stack, Role level, and Compensation.
  • Scans for initial visa keywords (Sponsorship, H1B, Transfer).

⚖️ Gate A: H1B Harsh Truth Gate (Agent 2 - Intel) Decision Point

Immediate validation using the USCIS Torch API. Strictly filters for H1B Transfer probability ≥ 0.7.

Validation Rules:
  • Checks historical LCA filings for the employer in Herndon, VA and Fairfax/Loudoun counties.
  • Verifies employer standing and previous sponsorship volume.
  • Go/No-Go: If probability < 0.7, the pipeline halts to save tokens and time.

📝 Phase 2: Resume Architect (Agent 3)

Claude 3.5 Sonnet + LaTeX pipeline. Enforces bolding of technical expertise based on JD requirements.

Advanced Tailoring:
  • Semantic search retrieves the most relevant Master CV bullets.
  • Bolding Engine: Uses dynamic regex to bold technical skills *only* if they appear in the JD.
  • ATS Simulation Loop: Recursive re-writes (Max 3) to hit target score ≥ 90%.

🤳 Gate B: Human-on-the-Loop Final Approval

Mandatory pause. System sends a Slack/WhatsApp summary for final verification before submission.

Review Package:
  • Summary of the Job, the company's H1B score, and the final tailored PDF.
  • Options: [✅ Approve & Submit] [✏️ Manual Edit] [❌ Discard].
  • Ensures no "hallucinated" details ever reach a recruiter.

🚀 Phase 3: Pilot & Networker

Final execution layer. Handles the RPA submission and post-apply LinkedIn outreach.

  • Agent 4 (Pilot): Fills forms via Browserbase/Anthropic "Computer Use".
  • Agent 5 (Networker): Identifies hiring stakeholders and sends a 1st-degree personalized connection.
Tier 3: Foundation Layer (Infrastructure)

🕵️‍♂️ Human Behavior Engine Stealth

Cross-cutting utility used by all browser agents. Bypasses detection via randomized emulation.

Anti-Bot Features:
  • Gaussian Jitter: Natural delays between clicks and keystrokes.
  • Bezier Pathing: Human-like mouse movements (No teleports).
  • Fingerprint Rotation: Residential proxies and unique UA strings.
  • Business Hour Gating: Activity limited to East Coast working hours.

🗄️ Data Layer (Polyglot Memory)

The system's persistent record. Ensures zero duplication and full auditability.

  • PostgreSQL: Job metadata, Application status, and Version tracking.
  • Object Storage (S3): Archives of every specific PDF generated.
  • Vector DB: Semantic storage for past interview answers and master bullets.
